# Release Process

1.  **Version Bump**: Update version in `Cargo.toml` workspace section.
2.  **Changelog**: Finalize `CHANGELOG.md` with a summary of changes since the last tag.
3.  **Tagging**: Create a signed git tag: `git tag -a vX.Y.Z -m "Release vX.Y.Z"`.
4.  **Draft GitHub Release**: Push the tag and draft a release on GitHub.
5.  **Artifact Generation**: Collect binaries for Linux and Windows (generated by CI or local build).
6.  **Publication**: Publish the release on GitHub and (optionally) `crates.io`.
